# nkigym IR Reference

## 1. Overview

The nkigym IR is a 3-layer system:

1. **`f_nkigym`** (Python function) ‚Äî the math-level description using `NKIOp` calls. Straight-line code: no loops, no conditionals. Written by hand or synthesized by an agent.
2. **`KernelModule`** (dataclass) ‚Äî the IR envelope. Contains signature, tensor/dim declarations, a schedule tree (`TreeIR`), and a dependency cache (`DepCache`). Created by `build_canonical_module(f_nkigym, input_specs)` and transformed by rewrite atoms.
3. **Rendered NKI source** (string) ‚Äî generated by `render(module)`. Emits a `@nki.jit` function with explicit loop nests and `nisa.*` ISA calls.

**Core design principle:** every buffer (HBM, SBUF, PSUM) is a first-class `Tensor` in the IR with explicit `location`, `shape`, and `dtype`. Every `NKIOp` maps 1:1 to one ISA call. No phase multiplexing, no hidden buffers, no auto-expansion.

## 2. The NKIOp surface

All `NKIOp` subclasses live in `nkigym/src/nkigym/ops/`. Each subclass declares:

- **`NAME`** ‚Äî ISA call name (e.g., `"nc_matmul"`).
- **`OPERAND_AXES`** ‚Äî maps operand slot name ‚Üí abstract axis tuple (e.g., `{"stationary": ("K", "M")}`).
- **`OUTPUT_AXES`** ‚Äî maps output name ‚Üí abstract axis tuple.
- **`OUTPUT_DTYPES`** ‚Äî optional per-output dtype override (defaults to first operand dtype).
- **`AXIS_ROLES`** ‚Äî per-abstract-axis role classification (`AxisRole.PARALLEL`, `AxisRole.SEQUENTIAL`, `AxisRole.ACCUMULATION`). Omitted axes default to `PARALLEL`.
- **`TILE_LIMITS`** ‚Äî hardware tile size per abstract axis (e.g., `{"P": 128, "F": 512}`).
- **`INPUT_OPERANDS`** ‚Äî read-only operand slots (land in `BodyLeaf.reads`).
- **`RMW_OPERANDS`** ‚Äî read-modify-write operand slots (land in `BodyLeaf.reads_writes`). Matmul's `dst` is the only RMW operand in the op set.
- **`RFACTOR_RECIPE`** ‚Äî `"rmw"` (matmul), `"slot"` (activation_reduce), or `None`.

| Class Name              | ISA Call              | Purpose                                              | File                        |
|-------------------------|-----------------------|------------------------------------------------------|-----------------------------|
| `NKIAlloc`              | `nl.ndarray`          | Declare tensor storage (HBM / SBUF / PSUM)          | `ops/alloc.py`              |
| `NKILoad`               | `nisa.dma_copy`       | HBM ‚Üí SBUF copy                                      | `ops/load.py`               |
| `NKIStore`              | `nisa.dma_copy`       | SBUF ‚Üí HBM copy                                      | `ops/store.py`              |
| `NKIDMATranspose`       | `nisa.dma_transpose`  | HBM ‚Üî SBUF transpose via DMA engine                  | `ops/dma_transpose.py`      |
| `NKIMatmul`             | `nisa.nc_matmul`      | Matrix multiply (Tensor Engine)                      | `ops/matmul.py`             |
| `NKITranspose`          | `nisa.nc_transpose`   | SBUF ‚Üî PSUM transpose via Tensor Engine              | `ops/transpose.py`          |
| `NKIActivation`         | `nisa.activation`     | Standalone activation (P, F axes)                    | `ops/activation.py`         |
| `NKIActivationReduce`   | `nisa.activation_reduce` | Fused activation + free-axis reduce               | `ops/activation_reduce.py`  |
| `NKITensorScalar`       | `nisa.tensor_scalar`  | Elementwise tensor-scalar op with broadcast          | `ops/tensor_scalar.py`      |
| `NKITensorCopy`         | `nisa.tensor_tensor`  | Elementwise copy or fused op                         | `ops/tensor_copy.py`        |
| `NKITensorReduce`       | `nisa.tensor_reduce`  | Reduce along an axis (scalar-engine, post-activation_reduce close) | `ops/tensor_reduce.py` |
| `NKIMemset`             | `nisa.memset`         | Fill tensor with constant value                      | `ops/memset.py`             |

**ClassVars usage:** The canonical builder reads `OPERAND_AXES` / `OUTPUT_AXES` to unify abstract axes into concrete dim ids. `AXIS_ROLES` drives legality predicates (e.g., `Reorder` cannot swap PAR and ACC loops on the same dim). `TILE_LIMITS` determines tile size per dim. `INPUT_OPERANDS` and `RMW_OPERANDS` partition operand slots into `BodyLeaf.reads`, `BodyLeaf.writes`, and `BodyLeaf.reads_writes`.

## 3. f_nkigym shape

An `f_nkigym` function is a straight-line sequence of `NKIOp` calls. No `for` loops, no `if` statements, no helper functions. Example (matmul_lhsT_rhs):

```python
from nkigym.ops import nkigym_kernel
from nkigym.ops.alloc import NKIAlloc
from nkigym.ops.load import NKILoad
from nkigym.ops.matmul import NKIMatmul
from nkigym.ops.memset import NKIMemset
from nkigym.ops.store import NKIStore
from nkigym.ops.tensor_copy import NKITensorCopy

@nkigym_kernel
def matmul_lhsT_rhs_nkigym(lhs_T, rhs):
    lhs_T_sbuf = NKIAlloc(location="sbuf", shape=(K, M), dtype="bfloat16")()
    rhs_sbuf = NKIAlloc(location="sbuf", shape=(K, N), dtype="bfloat16")()
    psum_acc = NKIAlloc(location="psum", shape=(M, N), dtype="float32")()
    sbuf_prod = NKIAlloc(location="sbuf", shape=(M, N), dtype="bfloat16")()
    hbm_out = NKIAlloc(location="hbm", shape=(M, N), dtype="bfloat16")()

    NKILoad()(src=lhs_T, dst=lhs_T_sbuf)
    NKILoad()(src=rhs, dst=rhs_sbuf)
    NKIMemset(value=0.0)(dst=psum_acc)
    NKIMatmul()(stationary=lhs_T_sbuf, moving=rhs_sbuf, dst=psum_acc)
    NKITensorCopy()(src=psum_acc, dst=sbuf_prod)
    NKIStore()(src=sbuf_prod, dst=hbm_out)
    return hbm_out
```

**Key traits:**

- `@nkigym_kernel` decorator wraps the function; `functools.wraps` preserves `__wrapped__.__globals__` for AST parsing.
- All non-parameter tensors declared via `NKIAlloc(location=..., shape=..., dtype=...)()` before use.
- Every `NKIOp` call is two-part: `OpClass(config_kwargs)(operand_kwargs)`. Config kwargs (if any) bind at construction time; operand kwargs bind at call time.
- Return value is an HBM output tensor (via `NKIStore`).

## 4. KernelModule ‚Äî the IR envelope

`KernelModule` (dataclass in `nkigym/src/nkigym/codegen/ir.py`) is the envelope IR. Fields:

- **`func_name`** (str) ‚Äî emitted kernel name.
- **`param_names`** (list[str]) ‚Äî signature order.
- **`return_name`** (str) ‚Äî name of the return tensor.
- **`tensors`** (dict[str, Tensor]) ‚Äî all named tensors, keyed by name.
- **`dims`** (dict[str, DimInfo]) ‚Äî all concrete dims, keyed by dim id.
- **`body`** (TreeIR) ‚Äî the schedule tree (list of `LoopNode | BodyLeaf`).
- **`dep`** (DepCache) ‚Äî per-scope dependency cache (lazy rebuild).

### Tensor (dataclass)

Fields:

- **`name`** (str) ‚Äî source-level variable name.
- **`dim_ids`** (tuple[str, ...]) ‚Äî concrete dim ids in operand order.
- **`shape`** (tuple[int, ...]) ‚Äî element sizes aligned with `dim_ids`.
- **`dtype`** (str) ‚Äî element dtype (e.g., `"bfloat16"`).
- **`origin`** (TensorOrigin) ‚Äî `"param"` (HBM input), `"intermediate"` (SBUF handoff), `"return"` (final output).
- **`location`** (Literal["hbm", "sbuf", "psum"]) ‚Äî memory tier. Params are always `"hbm"`.
- **`buffer_degree`** (dict[str, int]) ‚Äî multi-buffer degree per dim (defaults to 1).

### DimInfo (dataclass)

Fields:

- **`dim_id`** (str) ‚Äî concrete dimension id (e.g., `"d0"`).
- **`total_size`** (int) ‚Äî full size.
- **`tile_size`** (int) ‚Äî per-tile size.
- **`num_tiles`** (int) ‚Äî `total_size // tile_size`.

### LoopNode (dataclass)

Fields:

- **`dim_id`** (str) ‚Äî concrete dim id.
- **`trip_count`** (int) ‚Äî number of iterations.
- **`role`** (AxisRole) ‚Äî `PARALLEL`, `SEQUENTIAL`, or `ACCUMULATION`.
- **`children`** (list[LoopNode | BodyLeaf]) ‚Äî nested loops or leaves.
- **`reduce_op`** (str | None) ‚Äî reduce combinator for ACC loops (e.g., `"sum"`).
- **`name`** (str | None) ‚Äî canonical loop name (e.g., `"i_d0_0"`).
- **`pipeline_depth`** (int) ‚Äî software pipeline depth (default 1).

### BodyLeaf (dataclass)

Self-describing leaf carrying all metadata needed to render one ISA call. Fields:

- **`op_cls`** (type) ‚Äî the `NKIOp` subclass.
- **`reads`** (dict[str, str]) ‚Äî operand slot ‚Üí tensor name for read-only operands.
- **`writes`** (tuple[str, ...]) ‚Äî tuple of tensor names written.
- **`reads_writes`** (tuple[str, ...]) ‚Äî tuple of tensor names read+written (RMW operands).
- **`kwargs`** (dict[str, Any]) ‚Äî config kwargs (e.g., `op='square'`, `value=0.0`).
- **`axis_map`** (dict[str, str]) ‚Äî abstract axis ‚Üí concrete dim id.
- **`dim_role`** (dict[str, AxisRole]) ‚Äî per-dim role for this leaf.

**Note:** `reads`, `writes`, and `reads_writes` are disjoint. A name appears in at most one of these three fields per leaf.

## 5. TVM-style Buffer vs Allocate separation

The IR separates **tensor identity** from **allocation scope**:

- **`module.tensors[name]`** is the source of truth for what a named tensor *is* ‚Äî shape, dtype, location, dim_ids, buffer_degree, origin. Persists across rewrites unless an atom explicitly mutates it.
- **`NKIAlloc` BodyLeaf** is the source of truth for *where* storage is created in emission order. Its only kwarg is `tensor_name`; the emitter looks up shape/dtype/location via `module.tensors[tensor_name]`.

Atoms that change tensor identity (e.g., `MultiBuffer` on `buffer_degree`) mutate `module.tensors`. Atoms that change allocation scope (e.g., `ComputeAt` on an alloc leaf) move the `NKIAlloc` leaf. The two concerns never mix.

This design mirrors TVM's `Buffer` (identity) vs `Allocate` (scope) split. Every non-parameter tensor has exactly one `NKIAlloc` leaf, which dominates all reads/writes of that tensor in emission order.

## 6. Canonical build pipeline

`build_canonical_module(f_nkigym, input_specs)` (in `nkigym/src/nkigym/codegen/canonical.py`) constructs the initial `KernelModule`. Steps:

1. **AST-parse** the math function to an ordered list of raw parsed ops (`_ParsedOpRaw` records).
2. **Unify abstract axes** (P, F, K, M, N, ...) across ops into concrete dim ids (d0, d1, ...) via shape-based unification.
3. **Derive per-dim total size + tile size** from `input_specs` and per-op `TILE_LIMITS`.
4. **Tag each tensor's origin** (`param` / `intermediate` / `return`).
5. **Build the canonical 1N-per-dim forest** with phase leaves at the deepest point. Each `NKIOp` call produces one tree; nested loops iterate over `touched_dims` (the union of all axis roles mapped to concrete dims). Alloc leaves are emitted at the forest root in source order.
6. **Assign canonical loop names** `i_<dim>_<ordinal>` across each tree via `_assign_canonical_names`.

The resulting `KernelModule` has one `LoopNode` per `(dim, nest-position)` ‚Äî no trip-1 tile partners. Tiles-per-block arithmetic-intensity dials live in `Split`, not canonical form.

**Canonical tree structure:** each compute/copy op produces a tree `L(d0, L(d1, ... L(dn, leaf)))` where `{d0, d1, ..., dn}` are the `touched_dims` for that op (dims with any role in `leaf.dim_role`). Alloc leaves have no nesting ‚Äî they appear as direct children of the forest root.

## 7. validate_dataflow_ordering

`validate_dataflow_ordering(module)` (in `nkigym/src/nkigym/codegen/ir.py`) enforces dataflow legality in pre-order DFS emission order. Rules:

1. **NKIAlloc leaves must appear FIRST for their tensor.** A tensor name cannot appear in any prior leaf's `reads`, `writes`, or `reads_writes` before its `NKIAlloc` leaf. Allocation reserves storage; any prior touch would reference non-existent memory.

2. **Non-alloc leaves require allocated operands.** Every name in `leaf.reads ‚à™ leaf.writes ‚à™ leaf.reads_writes` must either be a parameter or have been allocated by a prior `NKIAlloc` leaf.

3. **Reads-after-writes.** Every name in `leaf.reads ‚à™ leaf.reads_writes` must be a parameter or already-written by a prior leaf (in `written` set).

4. **RMW finalization.** For a tensor T with any RMW writer (a leaf with T in `reads_writes`), every non-RMW read of T (a leaf with T in `reads`, not `reads_writes`) must appear after the LAST RMW write. Otherwise the consumer reads a partial-accumulation value.

   Example: matmul's `dst` is RMW. The accumulation loop writes `psum_acc` via `reads_writes`. The drain `NKITensorCopy(src=psum_acc, dst=sbuf_prod)` has `psum_acc` in `reads` ‚Äî this leaf must appear after all matmul iterations complete.

5. **Return-tensor produced.** Every tensor with `origin == "return"` must be in the `written` set by the end of the forest walk.

The validator is the single source of truth for dataflow legality. No op-specific carve-outs.

## 8. Rewrite atoms

All atoms live in `nkigym/src/nkigym/tune/`. Each atom is a `@dataclass(frozen=True)` with `is_legal(module) -> bool` and `apply(module) -> KernelModule` methods. Atoms group into three categories (TVM Domain / Placement / Execution):

### Domain (loop structure)

- **`Split`** (`split.py`) ‚Äî partitions a loop into outer √ó inner when factor divides trip count. Raises `AtomLegalityError` if factor does not divide (no tail sibling, no predication).
- **`Reorder`** (`reorder.py`) ‚Äî swaps loop order. Legality: subtree purity (same leaf set under both loops) + no role conflict (cannot swap PAR and ACC loops on the same dim).
- **`Fuse`** (`fuse.py`) ‚Äî merges nested loops into one flattened iterator. Not in default sampler (synthetic-dim leaks).

### Placement (computation scope)

- **`ComputeAt`** (`compute_at.py`) ‚Äî moves a leaf (or a whole tree) to a new loop scope. Regenerates inner loops from the leaf's `touched_dims`. Legality: dataflow ordering preserved.
- **`ReverseComputeAt`** (`reverse_compute_at.py`) ‚Äî consumer-side dual of `ComputeAt`. Moves the target loop's children into the leaf's scope.
- **`HoistInvariant`** (`hoist_invariant.py`) ‚Äî intra-own-tree LICM. Pulls a leaf out of an inner loop if that loop's dim is not in the leaf's `touched_dims`.
- **`RFactor`** (`rfactor.py`) ‚Äî fissions a reducer into outer-split + staging + close. Two recipes:
  - **`"rmw"`** (matmul): emits per-outer-iteration PSUM alloc, drain to SBUF slot, closing `tensor_reduce` over outer dim.
  - **`"slot"`** (activation_reduce): successive calls write distinct slots of a staging buffer, closed by `tensor_reduce`.
  
  `RFactor` with recipe `"slot"` has a render gap for 3D SBUF staging tracked in `docs/superpowers/plans/2026-05-09-first-class-buffers-and-rfactor-followups.md`.

### Execution (memory / pipeline)

- **`MultiBuffer`** (`multi_buffer.py`) ‚Äî increases `Tensor.buffer_degree[dim]` to enable double/triple buffering. Shape widening happens at emission time.
- **`SoftwarePipeline`** (`software_pipeline.py`) ‚Äî sets `LoopNode.pipeline_depth > 1`. Renderer emits prologue/body/epilogue when `depth > 1`.

**Note:** `DecomposeReduction` was deleted. `RFactor` is the unified factorization atom.

## 9. Rendering

`render(module)` (in `nkigym/src/nkigym/codegen/render.py`) orchestrates the lowering pipeline:

1. **`place_buffers`** ‚Äî compute buffer shapes + slot counts from the LCA walk. `required_tiles(inner)` from the LCA of all reads+writes for a tensor.
2. **`inject_multi_buffer`** ‚Äî slot-index expressions for buffers with `buffer_degree > 1`.
3. **`inject_software_pipeline`** ‚Äî prologue/body/epilogue emission for loops with `pipeline_depth > 1`.
4. **`emit_ops`** ‚Äî per-op_cls ISA call-site emitters. Each `NKIOp` subclass has a corresponding `_emit_<name>` function.
5. **`emit_source`** ‚Äî top-level forest walker. Produces the NKI source string.

**Tree-position allocation:** `NKIAlloc` leaves emit `<name> = nl.ndarray(<shape>, dtype=nl.<dtype>, buffer=nl.<location>)` at the leaf's tree position. The emitter looks up `module.tensors[tensor_name]` for identity; shape is computed from `required_tiles` (the LCA walk including the alloc leaf itself).

**Body emitters:** keyed by `op_cls.__name__` in `emit_ops.py`. Example: `_emit_NKIMatmul` ‚Üí `nisa.nc_matmul(dst, stationary, moving)`.
